       u.id AS user_id, u.name AS user_name,
       p.predicted_home, p.predicted_away

S3-T2 ● Medium Fullstack

CSRF-beveiliging op alle formulieren

Beveilig alle POST-formulieren met een CSRF-token om aanvallen van buitenaf te voorkomen.

›
User Story

"Als ontwikkelaar wil ik dat formulieren beschermd zijn tegen CSRF-aanvallen zodat de app veilig is."

Wat is CSRF?

Een Cross-Site Request Forgery-aanval trikt een ingelogde gebruiker in om zonder het te weten een formulier in te dienen. Bijv. een kwaadaardige website stuurt een POST-request naar jouw poule-app en verwijdert een poule. Met een CSRF-token voorkom je dit: het token zit alleen in jouw formulier, niet in het verzoek van de aanvaller.

Acceptatiecriteria
  • → Voeg hulpfuncties csrfToken() en verifyCsrf() toe aan includes/auth.php
  • → Elk formulier in het project heeft een hidden input met het token
  • → Bij een ongeldige token wordt het formulier geweigerd met een foutmelding
  • → Minimaal: login.php, register.php, create_pool.php, predictions.php
Implementatie Richting
// In auth.php toevoegen:
function csrfToken(): string {
    if (empty($_SESSION['csrf_token'])) {
        $_SESSION['csrf_token'] = bin2hex(random_bytes(32));
    }
    return $_SESSION['csrf_token'];
}

function verifyCsrf(): void {
    $token = $_POST['csrf_token'] ?? '';
    if (!hash_equals($_SESSION['csrf_token'] ?? '', $token)) {
        http_response_code(403);
        die('Ongeldige beveiligingstoken.');
    }
}

// In elk formulier:
// <input type="hidden" name="csrf_token"
//        value="<?= csrfToken() ?>">

Richting
// In index.php, na de DB-verbinding:
$stmt = $pdo->query(
    "SELECT value FROM settings WHERE `key` = 'last_sync'"
);
$lastSync = $stmt->fetchColumn();

if (!$lastSync || (time() - strtotime($lastSync)) > 3600) {
    include __DIR__ . '/sync_scores.php'; // hergebruik!
    $pdo->prepare(
        "INSERT INTO settings (`key`,`value`)
         VALUES ('last_sync', NOW())
         ON DUPLICATE KEY UPDATE value = NOW()"
    )->execute();
}

Statistieken dashboard met grafieken

Voeg een statistiekenpagina toe met visuele grafieken van je puntenprestatie over de wedstrijden heen.

›
User Story

"Als deelnemer wil ik een grafiek zien van mijn punten per wedstrijd zodat ik mijn prestaties kan analyseren."

Acceptatiecriteria
  • → Nieuwe pagina stats.php
  • → Staafgrafiek of lijndiagram van punten per gespeelde wedstrijd (gebruik Chart.js via CDN)
  • → Toon ook: totale punten, gemiddeld per wedstrijd, beste wedstrijd
  • → PHP geeft de data door aan JavaScript via json_encode()
Chart.js Richtlijnen

Zo ziet een Chart.js staafgrafiek er uit met de huidige stijl. Dit is het gewenste eindresultaat.

PHP → JS Data doorgeven
// In PHP:
$chartData = [];
foreach ($predictions as $pred) {
    $chartData[] = [
        'match' => $pred['home_team'] . '-' . $pred['away_team'],
        'points' => (int)$pred['points'],
    ];
}

// In HTML:

Deelbare ranglijst-link

Genereer een publieke, alleen-lezen link naar de ranglijst van een poule die je kunt delen zonder dat de ontvanger een account nodig heeft.

›
User Story

"Als beheerder van een poule wil ik een deelbare link kunnen genereren zodat familie en vrienden de ranglijst kunnen bekijken zonder in te loggen."

Acceptatiecriteria
  • → Voeg kolom public_token toe aan de pools tabel (VARCHAR 64, default NULL)
  • → De aanmaker kan op pool_detail.php een publieke link genereren
  • → Nieuwe pagina public_ranking.php?token=xxxx toont alleen de ranglijst (geen andere data, geen login nodig)
  • → De link kan ook worden ingetrokken (token wordt gewist)
  • → Geen persoonlijke e-mailadressen zichtbaar op de publieke pagina
Token Genereren
$token = bin2hex(random_bytes(32)); // 64-char hex string
$pdo->prepare(
    "UPDATE pools SET public_token = ? WHERE id = ? AND created_by = ?"
)->execute([$token, $pool_id, $user['id']]);

// Publieke URL:
$url = 'https://jouwesite.nl/public_ranking.php?token=' . $token;
